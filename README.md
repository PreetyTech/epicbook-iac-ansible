# EpicBook - AWS Deployment with Terraform and Ansible

This repository provisions AWS infrastructure with Terraform and deploys the EpicBook application with Ansible.

## What This Project Does

Terraform creates the entire AWS environment including VPC, subnets, internet gateway, route table, security groups for EC2 and RDS, an Ubuntu EC2 instance, an RDS MySQL instance, an Elastic IP, and automatically generates the Ansible inventory file from its outputs.

Ansible configures the server and deploys the application by handling base OS updates, SSH hardening, Nginx reverse proxy setup, Node.js and PM2 installation, full application deployment, and database schema and seed data import.

## Repository Structure

- terraform/aws/main.tf - All AWS resources
- terraform/aws/providers.tf - AWS provider config
- terraform/aws/variables.tf - Input variable definitions
- terraform/aws/outputs.tf - Exposes public IP, DB host, admin user
- terraform/aws/terraform.tfvars.example - Template, never commit real values
- ansible/ansible.cfg - Ansible defaults and vault config
- ansible/site.yml - Orchestration, runs roles in order
- ansible/inventory.ini.example - Inventory template
- ansible/group_vars/web/main.yml - Non-sensitive app variables
- ansible/group_vars/web/vault.yml.example - Secrets template
- ansible/roles/common - System updates and SSH hardening
- ansible/roles/nginx - Nginx install and reverse proxy config
- ansible/roles/epicbook - App clone, Node.js, PM2 startup

## Terraform File Responsibilities

providers.tf sets Terraform and AWS provider version constraints. variables.tf defines all input variables with types and descriptions. terraform.tfvars holds your local concrete values and must never be committed. main.tf creates all AWS resources and also writes ansible/inventory.ini automatically after apply using the EC2 public IP and RDS endpoint. outputs.tf exposes public_ip, db_host, and admin_user after apply.

## Ansible File Responsibilities

ansible.cfg sets the inventory path, vault password file location, and SSH connection defaults. site.yml calls roles in order: common then nginx then epicbook. group_vars/web/main.yml holds non-sensitive config including app repo, destination path, port, DB name and user. group_vars/web/vault.yml holds sensitive secrets like db_password and should be encrypted with ansible-vault. inventory.ini is auto-generated by Terraform after apply and should never be hand-edited.

## Important Things to Keep in Mind

Variable alignment between Terraform and Ansible is critical. These three values must be identical in both tools or the deployment will fail with MySQL access denied: db_user, db_name, and db_password.

The inventory.ini file is written by Terraform after every apply. If the EC2 instance is replaced and the IP changes, re-run terraform apply and the inventory updates automatically.

The SSH key must exist on your local machine before running terraform apply. Terraform creates the AWS key pair from ssh_public_key_path in terraform.tfvars and will fail if the .pub file is missing.

A stable re-run of the playbook should return changed=0. All tasks are written using state-aware Ansible modules so the playbook only makes changes when something is actually out of the desired state.

## Prerequisites

- AWS account with CLI configured via aws configure
- Terraform version 1.3 or higher
- Ansible
- Python 3 with virtual environment activated
- SSH key pair available on your local machine as a .pub file

## Credentials and Ignored Files

These files contain secrets and must never be committed to Git: terraform/aws/terraform.tfvars, ansible/.vault_pass, ansible/group_vars/web/vault.yml, and ansible/inventory.ini.

Example terraform.tfvars:
region = us-east-1
resource_prefix = epicbook
instance_type = t3.micro
admin_username = ubuntu
ssh_public_key_path = ~/.ssh/id_rsa_epicbook.pub
db_name = bookstore
db_user = admin
db_password = REPLACE_WITH_STRONG_PASSWORD

Example ansible/group_vars/web/main.yml:
app_repo: https://github.com/pravinmishraaws/theepicbook.git
app_dest: /var/www/epicbook
app_user: www-data
app_group: www-data
nginx_site_name: epicbook
app_port: 3000
db_user: admin
db_name: bookstore

Example ansible/group_vars/web/vault.yml:
db_password: REPLACE_WITH_STRONG_PASSWORD

Encrypt it after creating:
cd ansible
ansible-vault encrypt group_vars/web/vault.yml

Example ansible/inventory.ini:
[web]
EC2_PUBLIC_IP

[web:vars]
ansible_user=ubuntu
ansible_ssh_private_key_file=~/.ssh/id_rsa_epicbook
ansible_python_interpreter=/usr/bin/python3
db_host=RDS_ENDPOINT

## Step-by-Step Deployment Guide

Step 1 - Clone this repository:
git clone https://github.com/PreetyTech/epicbook-iac-ansible.git
cd epicbook-iac-ansible

Step 2 - Create SSH key pair:
ssh-keygen -t rsa -b 4096 -f ~/.ssh/id_rsa_epicbook -N 
chmod 600 ~/.ssh/id_rsa_epicbook
chmod 644 ~/.ssh/id_rsa_epicbook.pub

Step 3 - Configure Terraform variables:
cd terraform/aws
cp terraform.tfvars.example terraform.tfvars
Edit terraform.tfvars with your real region, SSH key path, and database credentials.

Step 4 - Initialize and apply Terraform:
terraform init
terraform validate
terraform plan
terraform apply
After apply confirm that ansible/inventory.ini was generated with the correct EC2 IP and that db_host matches the RDS endpoint.

Step 5 - Prepare Ansible secrets:
cd ../../ansible
cp group_vars/web/vault.yml.example group_vars/web/vault.yml
echo your-vault-password > .vault_pass
chmod 600 .vault_pass
ansible-vault encrypt group_vars/web/vault.yml

Step 6 - Verify SSH access:
ssh -i ~/.ssh/id_rsa_epicbook ubuntu@EC2_PUBLIC_IP hostname

Step 7 - Run the Ansible playbook:
ansible-playbook -i inventory.ini site.yml
First run deploys everything. Each subsequent run should return mostly ok with changed=0.

Step 8 - Open in browser:
http://EC2_PUBLIC_IP
Confirm the EpicBook bookstore loads with books, cart, and checkout working.

Step 9 - Confirm idempotency:
ansible-playbook -i inventory.ini site.yml
A clean stable run returns changed=0 across all tasks.

## Day-2 Operations

Update infrastructure:
cd terraform/aws
terraform plan
terraform apply
If EC2 is replaced and the IP changes, inventory.ini is regenerated automatically.

Redeploy application only:
cd ansible
ansible-playbook -i inventory.ini site.yml

Destroy all infrastructure:
cd terraform/aws
terraform destroy
Warning: this permanently removes EC2, RDS, and all data unless you have a backup strategy in place.

## Troubleshooting

Terraform key pair file not found: Ensure the .pub file exists at the path in terraform.tfvars before running apply.

SSH connection timeout: Confirm you are using the current IP from ansible/inventory.ini. Verify the EC2 security group has inbound TCP port 22 open. Confirm the private key matches the public key path used by Terraform.

MySQL access denied: Confirm db_user, db_name, and db_password are identical in terraform.tfvars and ansible/group_vars/web/vault.yml. Confirm the RDS endpoint in inventory.ini is correct.

Ansible changed on every run: Replace unconditional shell and command tasks with state-aware Ansible modules. Add when conditions to tasks that should only run once.
